算法 · 01

延迟 1/rmsinv_rms commute

inv_rms 是 per-row 标量，沿线性的 QKV / RoPE 可交换。把除法从归约后推到 rope 里一次标量乘 —— x·γ 不再等 sum-of-squares 归约，rms_recip 与 QKV proj 重叠。

恒等式 
# RMSNorm(x) = (x * inv_rms) * gamma,  inv_rms[b] = 1/sqrt(mean_k x^2 + eps)
# inv_rms 是 per-row 标量, 对 matmul 可交换:
#     q = inv_rms * ((x * gamma) @ Wq)
# 于是把 ÷rms 推迟到 QKV 之后, 折进 rope 一次标量乘
示例 · decode_fwd.py:444-460, 632-704 
# ── rms_recip: 一个独立 CORE_GROUP 任务, 只算 1/sqrt(mean(x^2)+eps) ──
with pl.at(level=pl.Level.CORE_GROUP, name_hint="rms_recip",
           deps=[prev_out_tids[i] for i in range(DOWN_ON)]) as rms_tid:
    partial_sq = pl.full([1, BATCH], dtype=pl.FP32, value=0.0)
    for kb in pl.pipeline(HIDDEN // RMSNORM_K_CHUNK, stage=4):
        k0 = kb * RMSNORM_K_CHUNK
        x_chunk = hidden_states[:, k0 : k0 + RMSNORM_K_CHUNK]
        partial_sq = pl.add(partial_sq,
            pl.reshape(pl.row_sum(pl.mul(x_chunk, x_chunk)), [1, BATCH]))
    variance = pl.reshape(pl.add(pl.mul(partial_sq, HIDDEN_INV), EPS), [BATCH, 1])
    inv_rms = pl.recip(pl.sqrt(variance))
    inv_rms_states = pl.assemble(inv_rms_states, inv_rms, [0, 0])
rope_dep_tids[RMS_TID_IDX] = rms_tid

# ...QKV 直接用 normed_in(= x*gamma) 开算, 完全不等 rms_tid...

# rope (fa_fused phase-0) 内才把 inv_rms 乘回 —— 一次 per-row 标量乘:
inv_rms_b = pl.read(inv_rms_states, [b, 0])
q_raw = pl.mul(q_proj_slab, inv_rms_b)   # 1/rms 折进这里, 而非归约处
算法 · 02

flash-decoding 块级负载均衡fa_work_build + grid-stride

先用 AIV 预处理把 ragged batch 的真实 seq-block 压成无缝 work table，再 core = w % NUM_CORES 轮询均分 —— 摆脱 per-batch 长度倾斜（≈1.25× 理想）。

示例 · decode_fwd.py:548-560, 724-734 
# ── AIV 预处理: 把真实 seq-block 压成 gap-free work table ──
with pl.at(level=pl.Level.CORE_GROUP, name_hint="fa_work_build") as work_tid:
    cursor = pl.read(seq_lens, [0]) * 0            # scalar 0 (INDEX)
    for wb in pl.unroll(BATCH):
        wb_ctx = (pl.read(seq_lens, [wb]) + (SEQ_TILE - 1)) // SEQ_TILE
        for wp in pl.range(wb_ctx):
            pl.tensor.write(fa_work_table, [cursor + wp, 0],
                pl.cast(wb * MAX_CTX_BLOCKS + wp, target_type=pl.INT32))
        cursor = cursor + wb_ctx                   # 前缀和游标
    pl.tensor.write(fa_total, [0, 0], pl.cast(cursor, target_type=pl.INT32))

# ── fa_fused 核内: 片上读块数, 每核 grid-stride 领等价代价块 ──
fa_total_blocks = pl.cast(pl.read(fa_total, [0, 0]), target_type=pl.INDEX)
for fa_w in pl.range(fa_core, fa_total_blocks, NUM_CORES):   # core = w % N
    fa_enc = pl.cast(pl.read(fa_work_table, [fa_w, 0]), target_type=pl.INDEX)
    fa_b = fa_enc // MAX_CTX_BLOCKS                # 还原 batch
    fa_p = fa_enc % MAX_CTX_BLOCKS                 # 还原 block

                        pl.row_expand_mul(oi_tmp_valid, beta))
            mi = mi_new
        ctx = pl.row_expand_div(oi, li)
        attn_out = pl.assemble(attn_out, ctx_flat_bf16, [os_b, os_q_base * HEAD_DIM])
算法 · 04

融合 dcr_xgamma1 register block → 2 outputs

同一寄存器 FP32 块同时产出层残差 out 与下一层的 x·γ normed_out —— 省掉 x_gamma 的整轮 GM 读写往返，两个 kernel 合一。

示例 · decode_fwd.py:1066-1090 
# dcr_xgamma: 一个 spmd(DOWN_ON) dispatch, DOWN_ON 个块天然并行
with pl.spmd(DOWN_ON, name_hint="dcr_xgamma",
             deps=[down_tids[i] for i in range(DOWN_ON * K_SPLITS)]) as dcr_tid:
    n_out = pl.tile.get_block_idx()
    n0 = n_out * DOWN_TN
    # OUTPUT 1: 层残差 (down_acc + post_norm, 均 FP32) -> out (残差流)
    out_chunk = pl.add(down_acc_all[:, n0 : n0 + DOWN_TN],
                       post_norm_partial[:, n0 : n0 + DOWN_TN])
    out = pl.assemble(out, out_chunk, [0, n0])
    # OUTPUT 2: 下一层 x*gamma, 同一寄存器块直出 (无 GM 回读 out)
    gamma_next = pl.slice(input_rms_weight, [1, DOWN_TN], [next_gamma_idx, n0])
    xg = pl.col_expand_mul(out_chunk, gamma_next)
    normed_out = pl.assemble(normed_out, pl.cast(xg, target_type=pl.BF16), [0, n0])

# 一个 dcr dispatch tid 同时承载 out + normed_out 的全部 DOWN_ON slab;
# 下一层的 rms_recip/seeds(读 prev_out_tids) 与 QKV(读 prev_normed_tids) 都等它
for _slab in pl.unroll(DOWN_ON):
    prev_out_tids[_slab] = dcr_tid
    prev_normed_tids[_slab] = dcr_tid
算法 · 05

融合 QK-norm + RoPEin-register norm→rotate

inv_rms 缩放 · gamma · per-row qk_inv 归一 · RoPE 旋转，全部在寄存器内一次完成 —— 省掉 q/k_proj_norm 中间张量的 GM 往返；零行填充只为 32B 对齐。

示例 · decode_fwd.py:655-679（K 路径） 
# 读原始 k_proj 行, 列补零到 K_RED_ROWS (col-major 32B 对齐), reshape 后 ×1/rms
k_raw = pl.mul(
    pl.reshape(pl.concat(k_proj[b : b + 1, kv_col : kv_col + HEAD_DIM], k_red_pad),
               [K_RED_ROWS, HEAD_DIM]),
    inv_rms_b)
k_ss  = pl.row_sum(pl.mul(k_raw, k_raw))                 # QK-norm 平方和
k_inv = pl.recip(pl.sqrt(pl.add(pl.mul(k_ss, HEAD_DIM_INV), EPS)))
k_normed = pl.row_expand_mul(pl.col_expand_mul(k_raw, k_norm_w), k_inv)  # ×γ ×qk_inv
k_full = k_normed[0:1, :]                                # 只保留 row 0 (head)
k_lo = k_full[:, 0:HALF_DIM]
k_hi = k_full[:, HALF_DIM:HEAD_DIM]
# RoPE (NeoX half-split) 旋转
rot_lo = pl.sub(pl.col_expand_mul(k_lo, cos_lo), pl.col_expand_mul(k_hi, sin_lo))
rot_hi = pl.add(pl.col_expand_mul(k_hi, cos_hi), pl.col_expand_mul(k_lo, sin_hi))
rot = pl.concat(rot_lo, rot_hi)                          # 合成一整行 -> 1x MTE3 写
cache_row = layer_cache_base + (wr_slot_block * NUM_KV_HEADS + ki) * BLOCK_SIZE + wr_slot_offset
k_cache = pl.assemble(k_cache, pl.cast(rot, target_type=pl.BF16), [cache_row, 0])
02

核内优化

intra-kernel · squeeze the cores

在单个 kernel 内部把算力压满：AIC/AIV 混合根、瘦高矩阵切 K、把 MTE2→cube/vec→MTE3 三条硬件流水线流起来，以及把一块张量在双 AIV lane 上拆算。

核内 · 01

mix kernelAIC + AIV 混合根

一个 kernel 内同时跑 AIC(cube) 与 AIV(vector)，两者用 tpush / tpop 就地握手同步（而非跨 kernel 经 AICPU）—— 既让 cube 与 vector 重叠，又把原本 rope / attn / online-softmax 三个 kernel 合并成一个，减少 AICPU 调度。三相之间用 syncall 做全核 barrier。

            for it in pl.pipeline(ROPE_ITEMS_PER_CORE, stage=2):
                ...                                # 写 k_cache / v_cache / all_q_padded
    pl.system.syncall(core_type="mix")             # 把 cache/q 发布给全核

    # ── phase-1: QK/SV (AIC) 与 softmax (AIV) 混合, C<->V 靠 tpush/tpop 握手 ──

核内 · 02

matmul 切 Ksplit-K + atomic Add

decode 的矩阵是 M(=BATCH 16) / N 小、K 大的瘦高形，光切 M/N 凑不满核。切 K 把 reduction 维分成多片，各片算部分和后用硬件原子加汇入同一零种子输出 —— 补足并行度、分片可跨核。QKV / out / gate / up / down 全用此形。

示例 · decode_fwd.py:1016-1034（down_proj） 
for n_out in pl.parallel(DOWN_ON):                 # N 轴只有 5 块 (太少)
    n0 = n_out * DOWN_TN
    for k_split in pl.range(K_SPLITS):             # -> 再切 17 个 K-slice 补并行
        k0 = k_split * DOWN_TN
        with pl.at(level=pl.Level.CORE_GROUP, name_hint="down_proj",
                   deps=[seed_tid, silu_tids[k_split]]) as down_tid:
            a0 = mlp_tile[:, k0 : k0 + DOWN_TK]
            w0 = w_down[layer_inter_base + k0 : layer_inter_base + k0 + DOWN_TK,
                        n0 : n0 + DOWN_TN]
            c_acc = pl.matmul(a0, w0, out_dtype=pl.FP32)         # 首片
            for lk in pl.pipeline(1, N_SUB_K, stage=2):         # 内层 16 个 K chunk
                ks_off = lk * DOWN_TK
                a_k = mlp_tile[:, k0 + ks_off : k0 + ks_off + DOWN_TK]
                w_k = w_down[layer_inter_base + k0 + ks_off : ..., n0 : n0 + DOWN_TN]
                c_acc = pl.matmul_acc(c_acc, a_k, w_k)          # 累加
            down_acc_all = pl.assemble(down_acc_all, c_acc, [0, n0],
                                       atomic=pl.AtomicType.Add) # 分片原子加
        down_tids[n_out * K_SPLITS + k_split] = down_tid        # 收集 tid

核内 · 03

pl.pipeline软件流水 · stage=k

声明式软件流水：让一个核内的三条硬件流水线 —— MTE2(load) → Cube/Vec(compute) → MTE3(store) —— 跨迭代重叠起来（第 i+1 次的 load 与第 i 次的 compute、第 i-1 次的 store 并行）。stage = 保持在飞的迭代数。（AIC/AIV 之间的重叠由 mix kernel 负责，不是 pipeline。）

签名 · 两种形式 
for i in pl.pipeline(N_ITERS, stage=4):        # 0..N-1, 保持 4 级在飞
    ...
for kc in pl.pipeline(1, QKV_K_CHUNKS, stage=2):   # 带起始偏移(首片在外)
    ...
示例 · decode_fwd.py:479-489（QKV split-K 内层，load/compute/store 流水） 
q_acc = pl.matmul(normed_in[:, q_k_base : q_k_base + TK],   # 首片
                  wq[..., n0 : n0 + TN], out_dtype=pl.FP32)
for kc in pl.pipeline(1, QKV_K_CHUNKS, stage=2):
    kk = q_k_base + kc * TK
    # 每次迭代: MTE2 载 normed_in / wq 分块 -> Cube 做 matmul_acc -> (末尾)MTE3
    # 编译器把第 i+1 次的 load 与第 i 次的 cube 重叠, 掩盖搬运延迟
    q_acc = pl.matmul_acc(q_acc, normed_in[:, kk : kk + TK],
                          wq[layer_hidden_base + kk : layer_hidden_base + kk + TK,
                             n0 : n0 + TN])
q_proj = pl.assemble(q_proj, q_acc, [0, n0], atomic=pl.AtomicType.Add)

03

调度优化

scheduling · offload the AICPU

这一层的共同目标是减少 AICPU 要调度的东西：fan-out 铺核、spmd 缩减任务数、manual_scope 省 tensormap 建依赖、syncall 合并 kernel、手动 ring buffer 控内存、allow_early_resolve 提前下发。

调度 · 01

pl.parallelindependent task fan-out

把互不依赖的 N-tile 展开成 N 个独立任务并行派发。out_proj / gate / up / silu / down 都以 N 维 fan-out 铺满核。

示例 · decode_fwd.py:868-891（out_proj: 10×5 = 50 个 atomic-add 任务） 
for n_out_proj in pl.parallel(N_SPLITS_OUT):          # 10 个 N-tile 独立任务
    n_op = n_out_proj * OUT_TN
    for k_split_out in pl.range(K_SPLITS_OUT):        # 每个再切 5 段 K
        k_op = k_split_out * OUT_TK
        with pl.at(level=pl.Level.CORE_GROUP, name_hint="out_proj",
                   deps=[out_seed_tid, attn_done_tid]) as out_tid:
            out_a0 = attn_out[:, k_op : k_op + OUT_INNER_TK]
            out_w0 = wo[layer_hidden_base + k_op : layer_hidden_base + k_op + OUT_INNER_TK,
                        n_op : n_op + OUT_TN]
            out_c_acc = pl.matmul(out_a0, out_w0, out_dtype=pl.FP32)
            for out_lk in pl.pipeline(1, OUT_N_SUB_K, stage=2):
                out_c_acc = pl.matmul_acc(out_c_acc, out_a_k, out_w_k)
            attn_proj_fp32 = pl.assemble(attn_proj_fp32, out_c_acc, [0, n_op],
                                         atomic=pl.AtomicType.Add)
        out_tids[n_out_proj * K_SPLITS_OUT + k_split_out] = out_tid

调度 · 04

pl.system.syncallin-core barrier · merge kernels

核心价值是"合并 kernel"：原本要靠多个独立 kernel + AICPU 在其间同步的阶段，改成一个 kernel 内用核内同步指令 (syncall) 分相 —— 省掉 AICPU 的跨 kernel 调度与等待。core_type="mix" 同步 AIC(oi) 与 AIV(mi/li)。满占用契约：NUM_CORES(24)=24 AIC+48 AIV=910B 全核，否则未到齐的核永不返回（507018）。

示例 · decode_fwd.py:717, 781-792 
# 三个逻辑阶段(rope / attn / online-softmax)本可各是一个 kernel;
# 合并进 fa_fused 后, 用两道核内 syncall 分相, 不再经 AICPU 跨 kernel 同步:
pl.system.syncall(core_type="mix")     # phase-0 -> phase-1 (发布 cache/q)

# ── FULL-OCCUPANCY 契约 ──
# NUM_CORES(24) group blocks == 24 AIC + 48 AIV == 910B 全核 => full-occupancy;
# 部分派发 (或 NUM_CORES 偏离物理 AIC 数) 会留核未达, 等待永不返回 => 507018 超时。

pl.system.syncall(core_type="mix")     # phase-1 -> phase-2 (fence attn/softmax)
为什么省 AICPU：N 个 kernel → 1 个 kernel，AICPU 的派发次数与跨 kernel 依赖同步都随之减少；核内 syncall 是硬件 barrier，不占 AICPU。

调度 · 06

allow_early_resolvecritical-path pre-dispatch

任务定义上的布尔开关（pl.spmd / pl.at）。打开后该任务把 dispatch_fanin 提前传播给下游消费者，让下游在生产者真正完成前就开始 pre-staging / 下发 —— 压缩关键路径延迟。沿 q_proj → fa_fused → out_proj → 下一层 这条链逐级打开。

示例 · pypto-lib PR #691（沿关键路径逐级打开）

with pl.spmd(Q_ON * QKV_OK, name_hint="q_proj",
             allow_early_resolve=True):          # q_proj -> fa_fused 提前 stage
    ...
with pl.spmd(NUM_CORES, name_hint="fa_fused",
             allow_early_resolve=True):          # fa_fused -> out_proj
    ...
with pl.at(level=pl.Level.CORE_GROUP, name_hint="down_proj",
           allow_early_resolve=True):            # down -> 下一层 rms/QKV
    ...
# 每个被标记的任务把 dispatch_fanin 传给下游, 使其 "在生产者真正完成前预下发"
